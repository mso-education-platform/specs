Name: Learning Platform

Stack: Next.js 15 App Router, TypeScript (strict), Tailwind CSS, shadcn/ui, tRPC, Prisma, PostgreSQL

Structure: the app lives in `app/`, specs live in `specs/` (generated by SpecKit)

Architecture: hexagonal — `domain` → `application` → `infrastructure`

Absolute rule: never import from `infrastructure/` into `domain/`.

Naming rule: file names, classes, variables, and symbols must remain in English (translations are limited to user-facing content, not code identifiers).

Commit conventions: `feat|fix|chore` + SpecKit Issue reference. Example: "feat: add auth flow, closes #12".

When asked to implement a spec, you must:

1. Always start by reading:
   - specs/[n]-[name]/spec.md
   - specs/[n]-[name]/checklists/plan.md
   - specs/[n]-[name]/checklists/tasks.md
   - .copilot/skills/spec-implementation.md

2. Before each task, read the corresponding skill in .copilot/skills/

3. Never implement multiple tasks at the same time
